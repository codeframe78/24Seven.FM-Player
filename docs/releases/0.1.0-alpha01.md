# 24Seven.FM Player 0.1.0-alpha01

Version code: 2
Release stage: Closed Alpha candidate

## Highlights

- Native playback for StreamingSoundtracks.com, 1980s.FM, Adagio.FM, Death.FM, and Entranced.FM.
- Primary-to-fallback stream handling through a service-owned Media3 player.
- Background playback, notification/lock-screen controls, audio focus, and headset/Bluetooth media controls.
- Tapping the media notification body returns to the existing player task while keeping play/pause as the direct action.
- Live title and artwork, adaptive Now Playing dashboard, five-station carousel, and persistent mini-player.
- Native Queue/History, station authentication, community chat, song search/requesting, requester attribution, and the verified SST optional request-message workflow.
- Signed-in favorite-track discovery, filtering, accessible availability stoplights, and explicit favorite-to-request confirmation across all five stations.
- Favorites keeps the list open after submission and surfaces the station's success, rejection, or indeterminate-confirmation result in a fixed request-status card.
- Title/album/artist/genre library search; stable station-order/play-state Library sorting; number, track-name, album, artist, genre, year, length, and play-state Favorites sorting; and responsive indexed handling of full 1,500-track VIP/RIP Favorites lists.
- Queue-aware request availability with exact accessible green `Request Now` and red `Track Recently Played` states plus fresh pre-submit validation.
- Memory-only SST last-ten request history, request readiness/wait status, and explicit membership presentation.
- A capability-aware More directory that opens exact verified public pages for all five stations in Android Custom Tabs, including Death.FM's RIP membership route.
- M18-certified StreamingSoundtracks.com behavior across playback, metadata/artwork, Queue/History, authentication, Chat, Favorites, requests/messages, listener activity, and secondary pages.
- M19-certified 1980s.FM behavior, including independent native sign-in, protected restoration, authenticated Favorites/Chat/request access, station-only logout, metadata/artwork, Queue/History, and trusted Games/Awards pages.
- M20-certified Adagio.FM behavior, including independent native sign-in, protected restoration, authenticated Favorites/Chat/request access, station-only logout, classical metadata/artwork, Queue/History, and trusted public pages.
- M21-certified Death.FM behavior, including independent native sign-in, protected restoration, authenticated Favorites/Chat/request access, station-only logout, sparse metadata/artwork, compact Queue/History, and the separate RIP membership browser route.
- M22-certified Entranced.FM behavior, including independent native sign-in, protected restoration, authenticated Favorites/Chat/request access, station-only logout, extended Queue/History, and narrowly scoped legacy ICY punctuation normalization.
- System light/dark themes, expanded two-pane layouts, explicit double-Back exit confirmation, and tested 2×-text reflow for navigation, growing station cards, and account headers across phone, Fold, and Tablet windows.

## Safety and traffic behavior

- Queue/History reads share a minimum 60-second interval and run only while Queue, Favorites, or Requests is selected.
- Chat reads share a minimum 30-second interval and chat history is not persisted.
- Song requests are explicit one-shot user actions and are never automatically retried.
- Favorite lists are fetched only on demand while signed in, retained only in memory, and never persisted by the app.
- Listener request activity is fetched only on demand for verified stations, retained only in memory, and performs no mutation.
- Secondary pages open only after an explicit tap and exact same-station HTTPS validation; app sessions are never copied into the browser.
- Credentials are submitted only to the selected station; retained sessions are encrypted with Android Keystore.
- No advertising, analytics, tracking, or developer-operated backend is included.

## Known limitations

- M17 Private Messages, sleep timer, Cast/audio-output selection, and diagnostics are not included. PM Inbox/Sent discovery worked, but a profile-originated delivery test was not received and has been reported to the site owner.
- Natural server-side session expiry was not forcibly induced during representative station certification.
- The upload identity and Play testing route are selected, but the candidate has not yet been uploaded or verified through Play delivery.
- Compact phone, medium tablet, expanded tablet, and landscape/freeform layouts are validated on API 35 plus the
  physical Android 16 Razr, with an additional folded API 35 Pixel Fold check. A mechanically changed physical
  half-open fold-hinge posture remains an Alpha test item.
