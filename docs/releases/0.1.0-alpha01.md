# 24Seven.FM Player 0.1.0-alpha01

Version code: 2
Release stage: Closed Alpha candidate

## Highlights

- Native playback for StreamingSoundtracks.com, 1980s.FM, Adagio.FM, Death.FM, and Entranced.FM.
- Primary-to-fallback stream handling through a service-owned Media3 player.
- Background playback, notification/lock-screen controls, audio focus, and headset/Bluetooth media controls.
- Live title and artwork, adaptive Now Playing dashboard, five-station carousel, and persistent mini-player.
- Native Queue/History, station authentication, community chat, song search/requesting, requester attribution, and the verified SST optional request-message workflow.
- Signed-in favorite-track discovery, filtering, accessible availability stoplights, and explicit favorite-to-request confirmation across all five stations.
- Queue-aware request availability with exact accessible green `Request Now` and red `Track Recently Played` states plus fresh pre-submit validation.
- Memory-only SST last-ten request history, request readiness/wait status, and explicit membership presentation.
- A capability-aware More directory that opens exact verified public pages for all five stations in Android Custom Tabs, including Death.FM's RIP membership route.
- M18-certified StreamingSoundtracks.com behavior across playback, metadata/artwork, Queue/History, authentication, Chat, Favorites, requests/messages, listener activity, and secondary pages.
- System light/dark themes, expanded two-pane layouts, accessibility semantics, and explicit double-Back exit confirmation.

## Safety and traffic behavior

- Queue/History reads share a minimum 60-second interval and run only while Queue, Favorites, or Requests is selected.
- Chat reads share a minimum 30-second interval and chat history is not persisted.
- Song requests are explicit one-shot user actions and are never automatically retried.
- Favorite lists are fetched only on demand while signed in, retained only in memory, and never persisted by the app.
- Listener request activity is fetched only on demand for verified stations, retained only in memory, and performs no mutation.
- Secondary pages open only after an explicit tap and exact same-station HTTPS validation; app sessions are never copied into the browser.
- Credentials are submitted only to the selected station; retained sessions are encrypted with Android Keystore.
- No advertising, analytics, tracking, or developer-operated backend is included.

## Known limitations

- M17 Private Messages, sleep timer, Cast/audio-output selection, and diagnostics are not included.
- Representative authenticated certification remains pending for 1980s.FM, Adagio.FM, and Death.FM.
- A stable release signing identity and distribution channel must be selected before this candidate is shared with testers.
- Physical fold-hinge and tablet validation remain outstanding; responsive previews/tests cover those window sizes.

See `PRIVACY.md`, `docs/alpha-testing.md`, and `docs/m11-validation.md` before distribution.
